Let’s talk about open source in government.

Open source is a set of widely used, governmentally legal, officially advised practices aimed at increasing security, efficiency and reusability, and quality through open development, using existing free, publicly licensed code and creating new public domain code.

The topic of adopting open source methods and ideas can be uncomfortable. It introduces a number of ideas that can feel counter-intuitive to people charged with important missions and entrusted with the protection of citizen’s data. But in many ways, embracing open source can be a strong tool in achieving these goals. This page is meant to introduce a number of ways that open source practices can be employed to make your systems more secure and effective.

Open source methods have been around for a long time. They are at the heart of most of the technologies that we depend on and feel most familiar with.

Officially advised

Over the last several years, there has been an increasing shift within government recognizing the benefits of open source. Many agencies, such as [example/link], [example/link], and [example/link] are clarifying that they allow and prioritize of the use of open source code and tools, while also mandating that some amount of new code be released into the public domain. Perhaps most significantly, the Office of Management and Budget has issued a draft source code policy that puts open and closed source code on equal footing.
